# BPEJ アセット抽出候補

## 目的

`pokeemerald-jp` のROMマップとカテゴリ別シンボルDBを参照し、現在の
Expansion側アセットと同一で、ROMから安全に切り出せる可能性が高い
非音声アセットを整理したものです。

全候補は `tools/data/bpej_asset_extraction_candidates.json` に記録しています。
この一覧だけではビルド参照を変更せず、元アセットも削除しません。

## 選定条件

候補は次の条件をすべて満たすものに限定しています。

- BPEJシンボルDBの `status` が `verified`
- `verification.status` が `verified`
- Expansion側の変換後データとの `byte_match` が `true`
- ROM offsetとsizeが確定済み
- 現在のExpansion側に対応する元ファイルが存在する

BPEJ ROMはSHA1
`d7cf8f156ba9c455d164e1ea780a6bf1945465c2` で再確認しています。

## 候補数

| カテゴリ | 件数 | 主な対象 |
|---|---:|---|
| title_screen | 5 | レックウザ、雲、ロゴの光、タイルマップ、パレット |
| opening | 28 | 著作権画面、オープニング各シーンの画像・パレット・タイルマップ |
| ui | 22 | メッセージ枠、カーソル、手持ち、バッグ、戦闘UIなど |
| pc_box | 113 | ボックスUI、カーソル、壁紙、パレット、タイルマップ |
| pokedex | 21 | 図鑑UI、分布図、検索画面など |
| item_icons | 442 | BPEJ内にありExpansion側と一致するアイコン画像・パレット |
| 合計 | 631 | GBA LZ77 548件、非圧縮83件 |

599件は1つのROM範囲と1つの既存アセットが対応します。32件はPCボックス壁紙の
複合パレットで、1つのROM範囲が複数の既存ソースに対応するため、個別ファイルを
直接置換せず、連結済みバイナリを読む専用wrapperが必要です。

## 最初に進めやすい対象

小さい単位で試す場合は、次の順番が安全です。

1. `title_screen` 5件
2. `ui` の共通カーソル、標準メニューパレット、手持ち画面、戦闘UI
3. `pokedex` 21件
4. `opening` 28件
5. `pc_box` の共通UIを先に行い、壁紙はセット単位で追加
6. `item_icons` はアイコン画像と共有パレットの参照関係を維持して小分けに追加

各項目には、ROM offset、size、圧縮形式、提案出力先、既存ファイル、検証方法、
ROM範囲のSHA1、未抽出状態を記録しています。圧縮済みデータはBPEJ ROM内の
GBA LZ77 streamをそのまま抽出し、ビルド時に対象ラベルだけ明示的なwrapperへ
切り替える方式が、現在の音声抽出方式に最も近い実装になります。

## 今回候補に含めないもの

次はROMマップ上に領域または代表アンカーがありますが、現時点では一括抽出対象に
しません。

- `partial`、byte不一致、size未確定の項目
- 日本語固有でExpansion側と画像内容が異なるタイトル・UI素材
- マップ、レイアウト、タイルセット、メタタイル、属性テーブル
- ポケモン・トレーナーの全スプライト群
- フォント、charmap、テキスト、イベントスクリプト
- cry、アニメーションメタデータ、ポインタテーブル

これらは親テーブル、参照関係、圧縮形式、要素数まで確定してから、別manifestで
扱う必要があります。

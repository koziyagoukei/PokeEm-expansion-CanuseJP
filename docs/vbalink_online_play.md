# VBALinkを使用したオンライン通信

このプロジェクトでは、PC間のオンライン通信手段として **VBALink LAN版** の使用を推奨します。
インターネット越しでは、ポート開放を必要としない **Tailscale経由の接続** を標準手順とします。

VBALinkとTailscaleは本プロジェクトとは別の第三者製ソフトウェアです。利用規約と配布元を確認し、各自の責任で使用してください。また、ROMファイルを他人へ送信・配布しないでください。

## 対応範囲

- 対戦
- 交換
- 通信コンテスト
- 対応している2～4人通信

通信できるのは、次の互換情報がすべて一致するROM同士だけです。

- Project ID
- Link Compat
- Data Layout

原作エメラルド、他のROMハック、異なる通信互換バージョンとは接続できません。参加者全員が、本プロジェクトから配布された**同じリリース版**を使用してください。

## 必要なもの

- Windows PC（参加者ごとに1台）
- 同じ版のVBALink LAN版
- Tailscale
- 各参加者が正規に用意した、同じリリース版のROM
- ゲーム内で通常セーブを行ったセーブデータ

VBALinkは、配布元の[VBALinkダウンロードページ](https://www.vbalink.info/download-gba-emulator.htm)にあるLAN版を使用します。無関係なダウンロードサイトから取得した実行ファイルは使用しないでください。

## 事前準備

### 1. VBALinkを準備する

1. 参加者全員が同じVBALink LAN版を展開します。
2. `Options > Joypad > Configure`から操作キーまたはコントローラーを設定します。
3. `Options > Emulator`に`Pause when inactive window`がある場合はオフにします。
4. ゲームを一度単独で起動し、通常のゲーム内セーブが読み込めることを確認します。
5. 確認後はVBALinkを終了します。

セーブステートは通信中の同期状態を正しく保存できない場合があります。通信前後の保存には、ゲーム内の通常セーブを使用してください。

### 2. ROMの互換性を揃える

参加者全員が同じリリース版を使用していることを確認します。デバッグメニューを利用できるビルドでは、通信互換情報の表示から次の値も確認できます。

```text
Link Compat: 1
Data Layout: 1
Variant: Frontier Hack
```

`Variant`は現在の接続拒否条件ではありませんが、異なる配布物を混在させないための確認情報として使用します。

## Tailscaleを使用する標準手順

### 1. 仮想LANを準備する

1. ホストと参加者のPCへ[Tailscale](https://tailscale.com/download)をインストールします。
2. 同じtailnetへ参加するか、ホストPCを参加者へ共有します。
3. 両者のTailscaleを接続状態にします。
4. ホストはTailscale画面で、自分の`100.x.y.z`形式のIPアドレスを確認して参加者へ伝えます。
5. 必要に応じて、参加者側で次のコマンドを実行して到達性を確認します。

```powershell
tailscale ping 100.x.y.z
```

Tailscaleが`relay`接続になっている場合は遅延が増えることがあります。通信が不安定な場合は`tailscale status`で接続方式を確認し、可能なら`direct`接続になるネットワークを使用してください。

### 2. VBALinkを接続する

ROMを開く前に、VBALink同士を接続します。

#### ホスト

1. VBALinkを起動します。
2. `Options > Link > Settings`を開きます。バージョンによっては`Ctrl+M`でも開けます。
3. `Network`を有効にします。
4. `Server`を選択します。
5. 参加人数を設定できる場合は、実際の人数に合わせます。
6. `Start`を押して待機します。
7. Windows Defender ファイアウォールの確認が表示された場合は、VBALinkの**プライベートネットワーク**通信を許可します。

#### 参加者

1. VBALinkを起動します。
2. `Options > Link > Settings`を開きます。
3. `Network`を有効にします。
4. `Client`を選択します。
5. ホストから伝えられたTailscale IPアドレスを入力します。
6. `Connect`を押します。

全員の画面に`Connected`と表示されるまでROMを開かないでください。

### 3. ゲーム内で通信を開始する

1. 全員が`Connected`になった後、同じリリース版のROMを開きます。
2. 全員がゲーム内で同じ通信施設へ移動します。
3. 対戦、交換、コンテストなど、全員が同じ通信形式を選びます。
4. 接続が完了するまで、早送り、巻き戻し、ポーズ、セーブステートの作成・読込を行わないでください。
5. 通信終了後はゲーム内で通常セーブを行います。

## 直接接続する代替手順

Tailscaleを使用しない場合は、ホスト側ルーターでVBALinkが使用するポートをホストPCへ転送し、参加者がホストのグローバルIPアドレスへ接続します。

VBALink LAN版では一般に`5738`番ポートが使用されます。ただし、使用する版にポート設定がある場合は、その画面に表示された値を優先してください。必要なプロトコルが不明な版ではTCPとUDPの両方を設定します。

1. ホストPCのローカルIPアドレスを固定します。
2. ルーターでVBALinkの待受ポートをホストPCへ転送します。
3. Windows Defender ファイアウォールでVBALinkを許可します。
4. ホストが`Server`を開始します。
5. 参加者はホストのグローバルIPアドレスを入力して接続します。

CGNAT、集合住宅の共用回線、携帯回線などではポート開放できない場合があります。その場合はTailscaleを使用してください。グローバルIPアドレスは不特定多数へ公開しないでください。

## 安定動作のための設定

- 可能なら有線LANを使用します。
- 参加者間の物理的な距離が近いホストを選びます。
- 大容量ダウンロード、動画配信、クラウド同期を停止します。
- 全員が同じVBALink版と同じROMリリースを使用します。
- VBALinkの速度スライダーは、最初は安定性優先の設定にします。
- フレームスキップ、早送り、巻き戻し、チートは無効にします。
- 通信中はPCをスリープさせたり、ネットワークを切り替えたりしません。
- 4人通信を試す前に、2人通信で接続を確認します。

## トラブルシューティング

### `Connected`にならない

1. ホストがROMを開く前に`Server`を開始したか確認します。
2. 参加者がホストのTailscale IPを入力したか確認します。家庭内IPやグローバルIPと混同しないでください。
3. `tailscale ping`でホストへ到達できるか確認します。
4. Windows Defender ファイアウォールでVBALinkが許可されているか確認します。
5. 全員のVBALinkを終了し、ホスト、参加者の順で起動し直します。

### 「このゲームとは つうしん できません」と表示される

ROMの通信互換情報が一致していません。原作ROMや別のROMハックは使用できません。全員が同じプロジェクトの同じリリース版を使用しているか確認してください。

### 接続後に待ち続ける、または切断される

- 全員が同じ通信形式を選んでいるか確認します。
- 早送り、フレームスキップ、セーブステートを無効にします。
- Tailscaleが`relay`になっている場合は、より低遅延な回線または直接接続を試します。
- VBALinkの`Faster`設定を使用している場合は、安定性優先へ戻します。
- 一度全員のVBALinkを終了し、接続からやり直します。

### セーブデータが読み込まれない

- 通常のゲーム内セーブを使用したか確認します。
- ROM名とセーブファイル名が一致しているか確認します。
- 別のVBALink版が異なるセーブフォルダーを参照していないか確認します。
- 元のセーブデータを上書きせず、バックアップを取ってから設定を変更します。

## 通信開始前チェックリスト

- [ ] 全員が同じプロジェクトの同じリリース版を使用している
- [ ] 全員が同じVBALink LAN版を使用している
- [ ] セーブデータをバックアップした
- [ ] Tailscaleでホストへ到達できる
- [ ] ホストが先に`Server`を開始した
- [ ] 全員が`Connected`になってからROMを開いた
- [ ] 早送り、巻き戻し、フレームスキップ、チートを無効にした
- [ ] 全員がゲーム内で同じ通信形式を選んだ
